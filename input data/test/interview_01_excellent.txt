4. **Monitoring and Scaling**: I’d instrument the pipeline with metrics for throughput, latency, and error rates, using Prometheus and Grafana. 5. **Schema Management**: I’d use a schema registry like Confluent’s or AWS Glue Schema Registry to enforce schema evolution and compatibility, which is critical for long-running pipelines.

One tradeoff I’d consider is whether to pre-aggregate data in the streaming pipeline or push that logic downstream. I’d lean toward pushing raw data to the lake and using materialized views or dbt models for aggregations, so we can iterate on the business logic without reprocessing the entire stream.
